## what is GH Flow?
GH Flow is a lightweight, branch-based workflow that supports teams and projects where deployments are made regularly. This guide is a reference to understand the workflow and how to use it effectively.

## Workflow
1. **Create a branch**: When you start work on a new feature or bug fix, create a new branch from the main branch. This keeps your work isolated and allows for easier collaboration.
2. **Make commits**: As you work on your branch, make commits to save your progress. Each commit should represent a logical unit of work, making it easier to review and understand the changes.
3. **Open a pull request**: Once your work is ready for review, open a pull request (PR) to merge your branch into the main branch. This allows others to review your code, provide feedback, and discuss any changes before merging.
4. **Review and merge**: Team members can review the pull request, suggest changes, and approve it. Once the PR is approved, it can be merged into the main branch. This ensures that only reviewed and approved code is added to the main branch, maintaining code quality and stability.
